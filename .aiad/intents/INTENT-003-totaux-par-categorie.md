# INTENT-003-totaux-par-categorie

**Auteur** : Maxime Le Gac
**Date** : 2026-05-26
**Statut** : done

---

## POURQUOI MAINTENANT
L'utilisateur a saisi des dépenses mais n'a pas de vision d'ensemble pour contrôler son budget global.

## POUR QUI
L'utilisateur unique de l'interface web.

## OBJECTIF
L'utilisateur voit le total par catégorie sur une page dédiée, sans quitter l'application.

## CONTRAINTES
- Pas de nouveau framework JS
- Données déjà en base — aucune migration de schéma requise

## CRITÈRE DE DRIFT
Si la page affiche autre chose que les totaux corrects par catégorie (ex : graphiques non demandés, données d'autres mois sans filtre explicite, totaux incorrects).
